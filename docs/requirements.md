ТЗ: ИИ-АГЕНТ ДЛЯ МОНИТОРИНГА ТОВАРОВ WINSTYLE В EXCEL

1) Цель
Постоянно обходить список категорий одного сайта (Winestyle), извлекать по каждому товару:
- текущую цену (если есть цена со скидкой и обычная — брать большую из двух; если есть только одна — брать её);
- рейтинг товара (значение, сформированное из отзывов);
- наличие товара (в наличии / нет).
Далее обновлять существующий Excel-файл в директории проекта: искать строку товара по ссылке и обновлять только указанные поля. Если ссылка товара в Excel не найдена — товар пропускать.

2) Входные данные
- Конфиг сайта `config/sites/*.yml`:
  - `category_urls` — список категорий одного сайта;
  - селекторы карточки и нужных полей (`product_card_selector`, `product_link_selector`, `price_*`, `rating_selector`, `in_stock_selector/out_of_stock_selector`).
- Переменные окружения (`.env`):
  - путь к Excel (`EXCEL_FILE_PATH`), имя листа, названия колонок;
  - сетевые/поведенческие параметры Playwright;
  - лимиты и задержки.

3) Выходные данные
- Обновлённый Excel-файл в проекте.
- Обновляются только колонки:
  - `current_price` (или имя из `EXCEL_PRICE_COLUMN_NAME`),
  - `rating` (или имя из `EXCEL_RATING_COLUMN_NAME`),
  - `in_stock` (или имя из `EXCEL_IN_STOCK_COLUMN_NAME`).
- Сопоставление выполняется по URL товара (нормализованному, без трекинговых параметров).

4) Правила обновления Excel
- Если ссылка найдена — обновить цену/рейтинг/наличие.
- Если ссылка не найдена — пропустить запись без ошибки выполнения.
- Если целевые колонки отсутствуют — создать их в строке заголовков.
- Колонка ссылки обязательна; если не найдена среди `EXCEL_URL_COLUMN_CANDIDATES`, запуск завершается ошибкой конфигурации.

5) Обход категорий
- Поддерживаются режимы пагинации:
  - `numbered_pages`
  - `next_button`
  - `infinite_scroll`
- Относительные URL нормализуются в абсолютные через `base_url`.
- Дубликаты ссылок внутри одного цикла обхода пропускаются.

6) Надёжность и антиблок
- Движки: `browser` (Playwright) и `http` (оставлен в проекте).
- Ротация User-Agent, опциональный прокси-пул, retries, джиттер.
- Сохраняется watchdog/cooldown-инфраструктура и перезапуск через `scripts/cooldown_watchdog.py`.

7) Запуск
- Единичный цикл: `python -m app.main run`
- Постоянный режим: `python -m app.main watch`

8) Ограничения
- Проект больше не использует Google Sheets, OAuth и локальный state/resume для бизнес-потока обновления данных.
- Сервис работает с одним сайтом за запуск, но со списком категорий этого сайта.
